# Package P0056B Function Design

## Package

`P0056B`

## Scope

Package-scoped weather proxy construction for the energy-market AI LABB data layer.

## New functions

### `run_p0056b_weather_proxies()`

Purpose: Run the full P0056B pipeline.

Inputs: Feature DB path, weather DB path, evidence directory.

Outputs: `P0056BResult` with status, row counts, and evidence paths.

Side effects: Writes `area_weather_features_hourly_v1` rows generated by P0056B and evidence files.

Reason: Package entry point consistent with prior diagnostics modules.

Tests: Covered indirectly by unit tests and package execution.

### `create_schema()`

Purpose: Create the P0056B wide output table if absent.

Inputs: SQLite connection.

Outputs: None.

Side effects: Creates DB table/indexes.

Reason: Keeps P0056B table ownership explicit.

Tests: Package execution verifies table can be created and populated.

### `area_proxy_selection()`

Purpose: Return deterministic source/proxy weights for every required area.

Inputs: None.

Outputs: List of area/source selection rows.

Side effects: None.

Reason: Centralizes area mapping and makes fallback status reviewable.

Tests: Unit tests verify coverage and fallback flags.

### `load_source_inventory()`

Purpose: Inspect local weather and consumption source coverage.

Inputs: SQLite connection with weather DB attached.

Outputs: Dictionary with source table coverage, location metadata, and P0056A coverage.

Side effects: None.

Reason: Required review/evidence for local source sufficiency.

Tests: Package execution evidence.

### `build_area_weather_rows()`

Purpose: Build wide hourly feature rows for all selected areas.

Inputs: SQLite connection with weather DB attached.

Outputs: List of output row dictionaries.

Side effects: None.

Reason: Deterministic transformation from local weather sources to P0056B table rows.

Tests: Unit test synthetic feature derivation.

### `replace_output_rows()`

Purpose: Replace rows generated by P0056B in the output table.

Inputs: SQLite connection and row dictionaries.

Outputs: None.

Side effects: Deletes and inserts P0056B rows.

Reason: Idempotent package reruns.

Tests: Package execution verifies DB row counts.

### `validate_area_weather()`

Purpose: Compute coverage, missingness, regional sanity checks, and SE3 consistency.

Inputs: SQLite connection.

Outputs: Validation dictionary.

Side effects: None.

Reason: Required package evidence and PASS/WARN decision support.

Tests: Package execution evidence and unit tests for deterministic subparts.

### `write_evidence()`

Purpose: Write required Markdown/CSV evidence files.

Inputs: Evidence directory and summary dictionary.

Outputs: Mapping from evidence name to path.

Side effects: Writes package-run files.

Reason: P0056B requires durable package evidence.

Tests: Package execution verifies files exist.

## Important unchanged functions

### `src.mac.services.weather_history.storage.compute_area_proxy_hourly()`

Reason for no change: Existing proxy computation remains the source-of-truth for prior area proxies. P0056B reads its outputs instead of changing ingestion or historical calculations.

### `src.mac.services.spotprice_model_diagnostics.p0054z.run_p0054z_weather_series()`

Reason for no change: P0054Z remains the previous SE3 climate-zone artifact. P0056B references it for SE3 consistency but does not mutate it.
