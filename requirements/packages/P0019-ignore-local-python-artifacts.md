## Decision summary

Add a repository root `.gitignore` for local Python artifacts that are generated by normal Mac tooling and tests.

## Solution model

Python tooling in this repository can create local virtual environments and bytecode caches during test and diagnostic runs. These files are local execution artifacts, not G2 source-of-truth content.

The repository should ignore:

```text
.venv/
__pycache__/
*.py[cod]
```

This keeps `git status` focused on source, package, evidence and generated deploy artifacts that are intentionally tracked.

## Current behavior

The repository has no root `.gitignore`. Running Python tools or tests can create untracked `.venv/` and `__pycache__/` paths that make bootstrap sync appear dirty.

## Problem

Local generated Python artifacts can block the package bootstrap sync rule even though they are not package content.

## Target behavior

`git status --short --branch` should remain clean after local Python bytecode caches or a root `.venv/` are present.
