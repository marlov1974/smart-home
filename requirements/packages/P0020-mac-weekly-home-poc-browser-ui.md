## Decision summary

- Add a Mac-local browser/server wrapper around the P0018 weekly home optimizer POC.
- The web UI must be reachable from a phone on the same LAN by binding to `0.0.0.0` when explicitly started that way.
- The service is trusted-local/LAN only and must not be exposed outside the local network.
- The browser UI is for manual POC inspection only and must not control live FTX, VP, Shelly or Home Assistant runtime.
- The server must use Python standard library only unless Codex stops and proposes a package-scope change.
- Default port should avoid the existing spot forecast service port 8080. Use `8081` unless implementation review finds a repo-local convention that requires another port.
- The P0020 web server must call/reuse the P0018 planner implementation rather than duplicating optimizer logic.

## Solution model

P0018 produces the weekly heat/PPM/RH POC plan from:

```text
week_number
current_ppm
current_house_temp
```

P0020 adds a thin browser-facing layer:

```text
phone/browser -> Mac HTTP server -> P0018 planner -> HTML/JSON output
```

Manual flow:

```text
1. Start server on the Mac:
   python3 -m src.mac.labs.weekly_home_optimizer_poc.server --host 0.0.0.0 --port 8081

2. Open from phone browser:
   http://<mac-lan-ip>:8081/

3. Enter:
   week
   ppm
   house temp

4. Inspect:
   weekly summary
   hourly table
   selected supply_pct curve
   PPM absolute curve
   heat_kWh curve/table
   RH policy/effect columns
```

The UI can be simple HTML generated by Python. It does not need JavaScript or charts in this package. Tables and summary cards are enough.

## Current behavior

- P0018 defines and/or implements a Mac POC for weekly heat/PPM/RH planning.
- The POC is terminal/test oriented.
- The operator cannot conveniently run or inspect the POC from a phone browser.
- P0017 spot forecast uses port 8080 in local testing; this package should avoid that default.

## Problem

Manual validation of the optimizer requires quick iteration from phone/tablet without SSH/Terminal. The operator needs a local web page that can trigger POC runs and show the resulting 168-hour plan.

## Target behavior

Add a POC HTTP server under the P0018 lab package or a clearly documented adjacent Mac lab path.

Preferred module command:

```text
python3 -m src.mac.labs.weekly_home_optimizer_poc.server --host 0.0.0.0 --port 8081
```

Acceptable alternative:

```text
python3 -m src.mac.labs.weekly_home_optimizer_poc --serve --host 0.0.0.0 --port 8081
```

Codex must choose and document the final command in `requirements/package-runs/P0020/design.md`.

### HTML endpoint

The root page shall be a browser form:

```text
GET /
```

It shall show input fields for:

```text
week
ppm
houseTemp
```

Submitting the form shall produce a result page with:

```text
- input summary
- plan summary
- 168-hour table or scrollable table
- link/button to JSON output
```

The table must include, at minimum, the key P0018 output columns if available:

```text
hour
weekday_hour
outdoor_temp_c
outdoor_rh_pct
spot_index
heat_need_kWh
heat_kWh
heat_soc_pct
heat_cost_weight
rh_weight
supply_pct
flow_lps
ppm_delta
ppm_absolute
rh_delta
total_cost
```

### JSON endpoint

Expose a JSON endpoint for tests and manual inspection:

```text
GET /api/weekly-home-poc?week=2&ppm=500&houseTemp=22
```

JSON response shall include:

```text
input
summary
hours
```

`hours` must contain 168 hourly objects for a valid plan.

### Health endpoint

Expose:

```text
GET /health
```

Response shall be compact JSON indicating healthy status.

### Error handling

Invalid inputs shall return HTTP 400 with human-readable HTML for browser requests and JSON error for API requests.

Examples:

```text
week missing/invalid
ppm missing/invalid
houseTemp missing/invalid
week not modelable by available P0018/P0017 data
```

## Browser/manual test target

The operator should be able to test from a phone browser using:

```text
http://<mac-lan-ip>:8081/
```

And direct query:


The server must bind to `127.0.0.1` by default for safety, but support `--host 0.0.0.0` for explicit LAN testing.
